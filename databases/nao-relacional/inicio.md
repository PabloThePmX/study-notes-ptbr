# Inicio

* O banco de dados não relacional tem seus dados armazenados de maneira não estruturada ou semi-estruturada, ou seja, não são armazenados em uma tabela. (Ex.: MongoDB)
  * Um dado semi-estruturado é algo que tem uma estrutura porém não tem regras estritas sobre o que pode ser armazenado nele, como um JSON. 
  * Dado não estruturado são arquivos brutos, como pdf, posts, etc que não seguem qualquer padrão ou qualquer regra.
* Banco não relacional é mais rápido, tem mais performance que o relacional.
  * Mas isso não diz que é melhor que o relacional, pois cada um é para um tipo diferente de aplicação.
* Cada registro do semi estruturado pode conter campos diferentes, bem como tipos de valores diferentes para cada propriedade de cada registro.
* Tem mínimas regras, diferente do relacional.
